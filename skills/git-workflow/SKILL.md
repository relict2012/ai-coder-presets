# Purpose

确保所有代码修改遵循统一的 Git 开发流程，避免未经验证的代码进入主分支。

## Use this skill when

- 开始开发任务
- 创建开发分支
- 提交代码
- 推送远程仓库
- 合并主分支
- 初始化空仓库

## Do

### 空仓库

如果仓库为空：

- 初始化 Git
- 创建 `.gitignore`
- 创建 `main`
- 完成首次提交
- 配置 CI（如 Jenkins）

### 开发前

- Pull 最新 `main`
- 全量编译
- 编译失败停止流程
- 创建规范开发分支

允许的分支：

- `feat/*`
- `fix/*`
- `docs/*`
- `config/*`

### 提交

提交前：

- 全量编译
- 自动过滤无关文件
- 使用规范 Commit Message

格式：

```
feat: 新增订单查询接口
fix: 修复分页异常
docs: 更新开发文档
config: 修改云配置
```

推送后：

- 通知开发者进行人工测试

### 合并

只有收到明确的"人工测试通过"确认后：

- 拉取最新 `main`
- 检查冲突
- 合并
- 删除开发分支
- 保存变更记录

## Never

禁止：

- 编译失败仍提交
- 未测试直接 Merge
- 提交缓存文件
- 提交密钥文件
- 使用无意义分支名
- 使用无意义 Commit Message

## Verify

确认：

- 编译通过
- Jenkins 已执行
- Commit Message 合规
- 分支命名合规
- 已收到人工测试通过确认（Merge 前）

## Output

每个阶段完成后反馈：

- 当前阶段
- 已完成操作
- 下一步建议
